### 3. Civil Evacuation Simulation Model 🏆
*Awarded "Best Early Stage Poster Award" at BMW Summer School 2022*
A comprehensive simulation project validating evacuation routes and strategies for civil planning. This project required handling complex spatial data and building robust simulation models to predict crowd behaviors and optimize flow during tsunami inundation scenarios.
* **Tech Stack:** NetLogo, Python, QGis, GIT
* **My Role:** I developed Python scripts to preprocess and map complex tsunami inundation gauge data into discrete time simulations, and optimized the road network structure for high simulation efficiency. * **Repository:** Private Repository - Code available upon request

### 4. Autonomous Self-Guided Kart (OS-EV3)
Developed the operating logic and control systems for a self-guided kart using Mindstorms NXT/EV3 hardware. This project involved low-level programming to manage sensor inputs and motor outputs, mirroring the fundamentals of vehicle telemetry and autonomous trackside operations.
* **Tech Stack:** C, MakeFile, Docker, GIT
* **My Role:** I co-developed a 2-layered software architecture using coroutines to concurrently manage sensor readings and motor actuation on a low-power processor. I implemented the core logic via a Deterministic Finite State Automaton (DFA) that managed circuit navigation and a custom gyroscope calibration routine requiring calculated wall collisions to reset sensor drift.
* **Repository:** https://github.com/DavidePerticone/os-ev3-project

### 5. Distributed Cloud Infrastructure (Clouds Project 2023)
Designed and deployed a fault-tolerant, distributed key-value storage backend for a scalable platform. This project highlights my ability to build scalable, highly available systems and my familiarity with modern deployment and containerization concepts. 
* **Tech Stack:** C++, gRPC/Protobuf, Docker, Git
* **My Role:** I engineered the distributed backend using a `Shardmaster` to dynamically partition key ranges and balance loads across joining/leaving servers. I also implemented proxy `Shardmanagers` and `Shardservers` utilizing primary-backup replication to process client RPCs while maintaining strict fault tolerance.
* **Repository:** Private Repository - Code available upon request
